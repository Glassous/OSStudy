1. 操作系统接口一般有哪几种？

答：操作系统接口主要包括以下三类：
（1） 命令行接口（CLI）：通过控制台终端输入特定文本指令与操作系统交互的形式（如 Bash、PowerShell）。
（2） 图形用户接口（GUI）：通过图标、菜单、窗口及鼠标键盘进行可视化交互的形式（如 Windows、macOS 桌面）。
（3） 系统调用（System Call）接口：是操作系统内核提供给用户程序的编程接口，供程序员在程序中调用系统提供的服务。

2. 什么是系统调用？

答：
• 系统调用是操作系统内核提供给程序员的应用程序接口（API）的底层物理实现。
• 它是为了让用户程序能够安全、受控地访问系统硬件和执行特权操作（如读写磁盘、创建进程等），由内核暴露出来的一组子程序入口。

3. 操作系统中一般提供哪些类型的系统调用？

答：操作系统一般提供以下 6 大类系统调用：
（1） 进程控制类：如创建/终止进程、等待事件等。
（2） 进程通信类：如创建通道、读写共享内存、发送/接收消息。
（3） 文件操作类：如创建、删除、打开、关闭、读写文件及目录。
（4） 存储管理类：如申请和释放物理内存。
（5） 设备管理类：如请求/释放设备，启动外设传输。
（6） 信息维护类：如获取/设置时间、系统版本及统计信息。

4. 简述系统调用的执行过程。

答：系统调用的执行过程如下：
（1） 用户程序准备系统调用参数。
（2） 执行访管指令（或特殊的软中断/陷入指令，如 8086 的 `INT 21H`，Linux 的 `sysenter` / `syscall`）。
（3） CPU 暂停当前程序，进入特权模式，从用户态（User Mode）切换为核心态（Kernel Mode）。
（4） 硬件根据系统调用号检索中断向量表，转去执行对应的系统调用服务程序。
（5） 服务程序执行完后，将 CPU 重新置为用户态，返回到用户程序的断点处继续执行。

5. API 和系统调用之间有什么关系？

答：
• 定位差异：系统调用偏向于底层和内核实现；API（应用程序接口）则偏向于上层规范与行业标准。
• 运行空间：系统调用是在内核空间实现的，其修改仅需符合内核规范；而标准 API（如 POSIX、Win32 API）是在用户空间（如 libc）实现的，需遵守通用的业内标准，通常不能随意改动。
• 实现关联：
  - 许多 API 的底层实现就是通过调用相应的系统调用来完成工作的（如 C 语言的 `printf()` 最终调用 `write()` 系统调用）。
  - API 与系统调用并不完全是一一对应的：有的 API 不依赖系统调用（如纯算法计算 API），有的 API 封装了多个系统调用，而一个系统调用也可能同时服务于多个 API。

6. 什么是 Shell？

答：
• 定义：Shell 是操作系统外层的命令解释器，既是连接用户和内核的桥梁，也是一种强大的脚本编程语言。
• 作为命令语言：交互式地接收用户输入的指令，进行词法与语法分析后调用相应的核心程序去执行。
• 作为编程语言：定义了变量、参数，并支持分支、循环等结构，允许编写脚本来实现任务的自动化。
• 作用：虽然 Shell 不属于操作系统内核，但它高度协调各程序的并发运行与输入输出重定向。

7. 常用的 Shell 有哪些？

答：常用的 Shell 包含以下几类：
（1） ash：占用系统资源最少的小型 Shell，只包含 24 个内部命令，使用相对受限。
（2） bash：Linux 系统默认的标准 Shell，内部包含 40 个常用命令，使用极其广泛。
（3） ksh：含有 42 条内部命令，最大的优点是与商业 UNIX 发行版的 ksh 保持高度兼容。
（4） csh / tcsh：语法类似 C 语言的 Shell，包含 52 个内部命令。
（5） zsh：功能最强大、命令最丰富（有 84 个内部命令）的 Shell 之一，支持高度自定义的主题和自动补全。
