1. 何谓死锁？死锁产生原因和必要条件是什么？

答：死锁是指多个进程因竞争资源而造成的一种僵局，若无外力作用，这些进程都将无法继续推进。
产生原因：
（1）竞争系统资源；
（2）进程推进顺序不当。
必要条件（必须同时满足）：
（1）互斥条件；
（2）请求和保持条件；
（3）非剥夺条件；
（4）循环等待条件。

2. 列出日常生活中的几个死锁的例子。

答：
• 交通堵塞中多辆车互相等待对方让路；
• 两人在独木桥上迎面相遇，互不相让；
• 拓展阅读中的“钥匙开锁”例子：两个线程各自持有一把钥匙并等待对方释放另一把钥匙。

3. 在一个死锁中，只包含一个进程是否可能？

答：不可能。死锁至少涉及两个或更多进程相互等待对方所占有的资源。单个进程无法形成循环等待。

4. 死锁和饥饿的主要区别是什么？

答：
死锁：多个进程相互等待对方释放资源，都无法继续执行。
饥饿：某个进程长期得不到所需资源，无法执行，但其他进程可能仍在执行。

5. 一台计算机共 8 台磁带机，由 N 个进程共享，每个进程最多要 3 台，请分析 N 为多少时不会有死锁？

答：每个进程最多需要 3 台，系统共有 8 台。要避免死锁，需满足：
N × (3 - 1) + 1 ≤ 8 
2N + 1 ≤ 8
2N ≤ 7
N ≤ 3.5
因此，当 N ≤ 3 时，系统不会发生死锁。

6. 有 R1（2个）、R2（1个）两类资源 and 两个进程 P1、P2，两个进程均以：申请 R1 ⟶ 申请 R2 ⟶ 申请 R1 ⟶ 释放 R1 ⟶ 释放 R2 ⟶ 释放 R1的顺序使用资源，求可能达到的死锁点，并画出此时 the 资源分配图。

答：死锁可能发生在：
• P1 占用 R1 并请求 R2；
• P2 占用 R2 并请求 R1。
此时资源分配图中存在循环等待环，形成死锁。

7. 解决死锁的方法有几种？哪种方法最容易实现？哪种方法使资源的利用率最高？

答：四种方法：预防死锁、避免死锁、检测死锁、解除死锁。
最容易实现：预防死锁（如有序分配策略）；
资源利用率最高：避免死锁（如银行家算法）。

8. 预防死锁的途径具体有哪些？

答：通过破坏四个必要条件之一：
（1）破坏互斥条件（不现实）；
（2）破坏请求和保持条件（如预先分配策略）；
（3）破坏非剥夺条件（剥夺已分配资源）；
（4）破坏循环等待条件（如有序分配策略）。

9. 下面关于死锁问题的叙述哪些是正确的？哪些是错误的？说明原因。
1）参与死锁的所有进程都占有资源。
2）参与死锁的所有进程总至少有两个进程占有资源。
3）死锁只发生在无关进程之间。
4）死锁可发生在任意进程之间。

答：
（1）“参与死锁的所有进程都占有资源” —— 正确。每个死锁进程至少持有一个资源且在等待其他资源。
（2）“参与死锁的所有进程总至少有两个进程占有资源” —— 正确。死锁至少涉及两个进程，每个至少持有一个资源。
（3）“死锁只发生在无关进程之间” —— 错误。死锁可发生在任意进程之间，无关或相关进程均可能发生。
（4）“死锁可发生在任意进程之间” —— 正确。只要满足四个必要条件，任何进程之间都可能发生死锁。

10. 如果在银行家算法例题中最后一步，把 P0 发出的请求向量改为 Request0(0，1，0)，系统是否能将资源分配给它？

答：需重新执行银行家算法安全性检查。
（1）若分配后系统仍处于安全状态，则可分配；
（2）否则不可分配。
根据书中例题数据，当分配 Request0(0, 1, 0) 后，可用资源 Available 变为 (1, 5, 2)。此时进行安全性检查，无法找到一个安全序列使得所有进程顺利完成，因此系统处于不安全状态，不能将资源分配给它。

13. 试简化图 5-6 中的资源分配图，并利用死锁定理给出结论。

答：根据死锁定理，简化资源分配图的步骤如下：
（1）寻找一个非阻塞且非孤立的进程节点，若其申请的资源可以得到满足，则释放其占有的全部资源，使之成为孤立节点。
（2）重复此过程，直到找不到满足条件的进程。
（3）若最终所有进程节点都成为孤立节点，则该状态无死锁；若仍存在非孤立节点，则系统中存在死锁。
结论：经简化后，图中仍存在循环等待的非孤立节点，故系统当前存在死锁。
