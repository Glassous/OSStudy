1. 什么是操作系统？为什么学习操作系统？它的重要性在什么地方？

答：操作系统是最基本的系统软件，是一组有效管理和控制计算机系统的硬件和软件资源，合理地对各类作业进行调度以组织和控制系统工作流程，并方便用户使用计算机的程序的集合。操作系统被称为计算机之魂，它连接硬件与软件、用户与机器，决定了计算机的性能、稳定性和安全性。
学习操作系统不仅能帮助人们理解计算机的底层运行机制，更能提升编程能力、系统设计思维和问题解决能力，是计算机科学与技术领域不可或缺的核心知识。无论是开发应用程序、设计系统架构，还是研究前沿技术，操作系统的原理和思想都将成为你进阶的 “内功心法”。

2. 操作系统的发展经历了哪几个主要阶段？

答：操作系统的发展经历了：
• 单道批处理系统
• 多道批处理系统
• 分时系统
• 实时系统
• 通用操作系统
• 网络操作系统
• 分布式操作系统等阶段。

3. 从资源管理的角度，操作系统主要研究哪些问题？

答：从资源管理的角度，操作系统的主要功能就是对处理器、存储器、I/O 设备以及文件（数据和程序）这四类资源进行有效的管理。
• 处理器管理：负责处理器的分配与控制；
• 存储器管理：负责内存的分配与回收；
• I/O 设备管理：负责 I/O 设备的分配、回收与操纵；
• 文件管理：负责文件的存取、共享和保护。

4. 操作系统追求的目标是什么？

答：操作系统追求的目标包括：
• 方便性
• 有效性
• 可扩展性
• 开放性。

5. 操作系统的主要功能有哪些？

答：操作系统的主要功能包括：
• 处理器管理功能
• 存储器管理功能
• 设备管理功能
• 文件管理功能
• 接口管理功能。

6. 批处理有几种类型，都促进了计算机系统哪些方面的发展？

答：批处理主要有单道批处理系统和多道批处理系统两种。
（1） 单道批处理系统主要是为了解决人机矛盾以及 CPU 与 I/O 设备速度不匹配的矛盾，它的出现推动了计算机系统在资源利用、自动化管理、软硬件协同等多个关键领域的突破，为后续操作系统的发展奠定了基础。
（2） 多道批处理系统采用了多道程序设计技术，它允许多个程序同时进入内存，交替在 CPU 中运行。多道批处理系统不仅提高了 CPU 的利用率，同时也改善 I/O 设备和内存的利用率，从而提高了整个系统的资源利用率和系统吞吐量，最终提高了整个系统的效率。

7. 何谓联机 I/O？何谓脱机 I/O？

答：
• 联机 I/O：指 I/O 设备直接与主机连接，数据的输入输出过程由主机（通过操作系统或监控程序）实时控制和管理的模式。在这种模式下，I/O 设备与主机共享系统资源，操作过程与主机的运行紧密耦合。
• 脱机 I/O：指 I/O 设备不直接由主机控制，而是通过独立的外围处理机（或专用设备）完成数据的输入输出，再将处理后的结果“批量”传递给主机的模式。在这种模式下，I/O 操作与主机的运行过程分离，二者通过中间存储介质（如磁带、磁盘）间接交互。

8. 为什么引入多道程序（设计）？

答：操作系统中引入多道程序设计的核心目的是解决单道程序设计中“资源利用率极低”的根本问题，通过让多个程序“并发”运行，最大化 CPU、内存、外设等硬件资源的使用效率，从而提升整个计算机系统的吞吐量和处理能力。

9. 是什么原因推动了分时系统的发展，其主要特点是什么？

答：为了满足人们对于人机交互工作方式的需求，分时系统应运而生。
分时系统的主要特点包括：
• 同时性：允许多个终端用户同时使用系统；
• 独立性：用户彼此独立，互不干扰；
• 交互性：用户可直接与系统对话，及时调试；
• 共享性：多个用户共享系统软硬件资源；
• 及时性：系统能在极短时间内对用户的请求进行响应。

10. 实时系统的特点是什么，它与分时系统的相似和不同之处在哪里？

答：实时系统的特点包括实时性、操作的可预计性、人机对话的简单性、高安全与高可靠性。
实时系统与分时系统的比较：
• 相似之处：二者都支持多任务并发与资源共享。
• 主要差别：
  - 交互能力：分时系统的交互能力较强；而实时系统大都是具有特殊用途的专用系统，其交互能力受到一定的限制。
  - 响应时间：分时系统的响应时间一般都是以人能接受的时间延迟来确定的，其响应时间一般在秒数量级；而实时系统的响应时间则要求视应用场合而定，主要以控制对象或信息处理过程所能接受的时间延迟来确定，可能是秒级，也可能是毫秒级，甚至是微秒级。

11. 实时系统的主要应用对象是什么？

答：实时系统的核心特征是能在严格的时间约束内响应外部事件并完成处理，其应用对象主要集中在对“时间确定性”和“可靠性”要求极高的场景，例如：
• 工业控制与智能制造
• 交通运输与智能出行
• 军事与国防领域
• 医疗设备与生命支持
• 嵌入式与物联网场景
• 金融交易与实时数据处理等。

12. 处理器管理有哪些功能？它们的主要任务是什么？

答：进程管理功能主要包括：
• 进程控制功能：完成进程创建、撤销进程、唤醒进程、阻塞进程等操作；
• 进程同步功能：完成多个进程（含线程）运行的互斥与协调；
• 进程通信功能：用来实现进程之间的信息交换；
• 进程调度功能：从进程的就绪队列中选出一个进程把处理器分配给它，并为它设置运行现场使其投入运行。

13. 存储器管理有哪些功能？它们的主要任务是什么？

答：存储器管理的主要功能包括内存分配和回收、内存保护、地址映射和内存扩充等。
（1） 内存分配的主要任务：为每道程序分配内存空间，使它们“各得其所”；提高存储器的利用率，尽量减少不可用的内存空间（碎片）；允许正在运行的程序申请附加的内存空间，以适应程序和数据动态增长的需要。
（2） 内存保护的主要任务：确保每道用户程序都仅在自己的内存空间内运行，彼此互不干扰；不允许用户程序访问操作系统的程序和数据，也不允许用户程序转移到非共享的其他用户程序中去执行。
（3） 地址映射的任务：在 CPU 执行程序过程中把程序的逻辑地址转换为物理地址。
（4） 内存扩充的任务：借助于虚拟存储技术，从逻辑上扩充内存容量，使系统能够提供比物理内存大的容量。

14. 设备管理有哪些功能？它们的主要任务是什么？

答：设备管理的功能主要包括设备分配、缓冲管理和设备处理。
（1） 设备分配的基本任务：根据用户进程的 I/O 请求、系统当前的资源情况，按照某种设备分配算法为用户进程分配所需的设备；设备使用完后，系统应立即回收，以分给其他进程使用，提高设备的利用率。
（2） 缓冲管理的主要任务：缓和 CPU 与 I/O 设备间速度不匹配的矛盾，提高 CPU 与 I/O 设备之间以及各设备之间的并行性。
（3） 设备处理的基本任务：实现 CPU 与设备控制器之间的通信。接收由 CPU 发来的 I/O 命令（如读/写命令），转换为具体要求后传给设备控制器以启动设备；同时也将设备控制器的状态和信号传送给 CPU（如是否完好、是否准备就绪、I/O 操作是否已完成等），并在设备完成具体的 I/O 操作后及时响应中断，进行相应的中断处理。

15. 文件管理有哪些功能？它们的主要任务是什么？

答：文件管理的功能包括：
（1） 文件存储空间的管理：负责为每个文件分配必要的外存空间，提高外存利用率，并有助于提高文件系统的速度。
（2） 目录管理：为每个文件建立目录项，并对众多目录项进行有效组织。
（3） 文件的读、写管理：根据用户的请求，从外存中读取数据或将数据写入外存。
（4） 文件的存取控制：防止未经核准的用户存取文件，防止冒名顶替存取文件，防止以不正确的方式使用文件。

16. 什么是微内核操作系统？它具有哪些优点？

答：微内核操作系统是一种以“精简内核”为核心理念，将传统操作系统的大部分功能移至用户态实现的架构设计。
• 核心思想：仅保留操作系统最基础、最关键的功能在内核态（微内核），而将文件系统、设备驱动、进程间通信等非核心功能作为独立的“用户态服务进程”运行，通过内核提供的标准化接口实现服务间的交互。
• 优点：高可靠性、高扩充性、高移植性，以及适宜于分布式处理的计算环境等。
