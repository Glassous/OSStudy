1. 什么是数据项、记录和文件？

答：
• 数据项（Item）：又称为域（Field），是文件系统中基本的数据单元。一个域包含一个值，如学生的名字、入学日期，或者是传感器读取的数值。
• 记录（Record）：是一组相关域（数据项）的集合，用于描述一个对象在某方面的属性，可视为应用程序的一个基本处理单元。
• 文件（File）：是一组相关记录的集合，它被用户和应用程序视为一个逻辑实体，并可通过文件名进行访问。如学生登记表是一个文件，每条记录是一个学生的信息。

2. 什么是文件管理系统？

答：
• 从用户角度看：文件管理系统是操作系统的重要组成部分，允许用户创建、删除、读写和控制文件，并在不需要了解底层外存物理存储结构的前提下，方便且安全地管理数据集。
• 基本功能要求：包括实现对文件的按名存取、文件的目录管理、文件的共享与保护、外存空间的合理分配与回收，以及维护文件内容的完整性与一致性。文件管理系统由系统实用程序及管理模块组成，通常作为具有特权的系统服务运行。

3. 简述堆文件、顺序文件、索引顺序文件和索引文件的组织方式。

答：
（1） 堆文件（Pile）：
• 数据按到达的先后顺序进行收集并累积。每条记录由多个数据域组成，且记录的域可以不同，或者域相似但顺序不同，没有固定格式。主要目的仅仅是保存原始大量数据。
（2） 顺序文件（Sequential File）：
• 最常用的文件形式，每条记录使用固定的格式（等长记录或变长记录）。所有记录按照某个特定的关键域（Key Field）的顺序排列。由于每个域的长度和位置固定，查找时需要按照关键域的物理位置顺序扫描。
（3） 索引顺序文件（Indexed Sequential File）：
• 保留了顺序文件的关键特征（记录按关键域顺序组织），但增加了两项设计：一是对记录进行分组，二是对每组的首记录建立索引表项。索引提供了快速接近目标记录的查找能力，组内则进行顺序定位，大大加快了检索速度。
（4） 索引文件（Indexed File）：
• 为主文件中的每个记录都建立一个专门的索引表项。因此，主文件中的记录存储位置不再受限，可以随机存放，非常适合存储变长记录。查找时先检索索引表，再通过指针直接定位目标记录。

4. 试比较顺序文件和索引顺序文件的查找速度。

答：以查找包含 100 万条记录的文件为例：
（1） 顺序文件：要查找某个特定关键字记录，在最坏/平均情况下需要从头开始逐条比对。平均需要访问 50 万条记录，查找速度慢。
（2） 索引顺序文件：如果建立一个包含 1000 个索引项的索引表（每个索引项指向一个包含 1000 条记录的组）。查找时，平均只需在索引表中检索 500 次定位到对应的组，再在该组的主文件中顺序检索 500 次。平均只需进行 1000 次盘块/记录访问，查找开销从 50 万次降低到 1000 次，检索速度大幅提升。

5. 什么是文件目录？什么是两级目录和树形目录？

答：
• 文件目录：是一种用于标识系统中的文件属性及其物理地址的数据结构，实现“按名存取”的转换。目录自身也是一个文件，由操作系统进行统一管理与存取。
• 两级目录：将目录分为两级。第一级为主文件目录 MFD（Master File Directory），记录系统中各个用户文件目录的存放位置；第二级为用户文件目录 UFD（User File Directory），记录该用户自己创建的所有文件及属性。这解决了多用户文件同名冲突的问题。
• 树形目录（多级目录）：多用户系统中最普遍的结构。主目录被称为根目录，下面可创建多级子目录，数据文件作为树叶，目录作为树的分支结点。每个文件或子目录只能有一个父目录，通过唯一的路径名进行检索，检索速度快，便于文件分类与共享。

6. 绝对路径和相对路径有何不同？

答：
• 绝对路径名（Absolute Path Name）：是指从树形目录的根目录（“/”）开始，把沿途经过的全部子目录名和最终目标文件名依次连接起来构成的完整路径名。它在系统中的指向是唯一且绝对的。
• 相对路径名（Relative Path Name）：是指以当前工作目录（Current Directory）作为参考起点，到目标文件为止所构成的路径名。它缩短了路径长度，简化了日常输入。

7. 试比较外存分配方式的顺序分配、链接分配和索引分配。

答：
（1） 连续分配（顺序分配）：
• 特征：为每个文件分配一组物理上连续的磁盘块。
• 优点：顺序访问极快，磁头移动距离短，文件分配表表项极其简单（只需起始块号和块数）。
• 缺点：会产生大量的外部碎片，需要定期整理紧凑，且文件大小难以动态增长，需预先声明大小。
（2） 链接分配：
• 特征：以磁盘块为单位分配，每个物理块内包含指向下一个物理块的指针，呈链表结构。
• 优点：消除了外部碎片，不需要碎片整理，外存利用率高，文件可动态增长。
• 缺点：只适合顺序访问，不支持高效的随机直接访问。且指针占用物理块空间，若链中某指针损坏会导致文件断裂丢失。
（3） 索引分配：
• 特征：为每个文件分配一个专门的索引块，该索引块中按顺序记录了该文件所占用的所有物理盘块号。
• 优点：消除了外部碎片，且同时支持高效的顺序访问和直接随机访问，是目前最普遍的文件分配形式。
• 缺点：索引块本身会带来额外的数据开销。对于小文件，其索引块开销占比相对较大。

8. 在索引分配中为什么要引入多级索引？

答：
（1） 限制突破：单个索引块的大小是固定的（例如 1 KB 或 4 KB）。每个物理块号占一定字节（如 4 字节），这意味着单个索引块所能记录的物理块号个数（即文件大小）是有上限的。
（2） 大文件支持：为了能支持超大文件，系统需要引入多级索引（如两级索引、三级索引）。通过让主索引块指向一级索引块，一级索引块再指向二级索引块，最后二级索引块指向实际的数据块，使得文件的最大容纳物理空间呈指数级增长（如最大可支持几百 GB 到几 TB 的单文件）。

9. 请分析位示图方式的优缺点。

答：
• 定义：位示图是利用二进制的一位（bit）来表示磁盘中一个盘块的使用情况（通常 0 表示空闲，1 表示已分配）。所有盘块对应的二进制位集合构成了位示图。
• 优点：
  - 容易寻找相邻的连续空闲盘块（只需查找连续为 0 的二进制位即可）。
  - 位示图占用空间极小，可以完全常驻内存，盘块分配与回收操作可在内存中极速完成，显著减少磁盘的 I/O 启动频率。
• 缺点：
  - 磁盘容量巨大时，位示图也会相应增大，占用较多内存。
  - 在分配与回收盘块时，需要在物理盘块号与位示图的“行、列、位”三维坐标之间进行复杂的数学计算转换，耗费一定 CPU 算力。
