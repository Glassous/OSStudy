1. 处理机调度算法的共同目标是什么?批处理系统的调度目标又是什么?

答：处理机调度算法的共同目标和批处理系统的调度目标如下：
一、 处理机调度算法的共同目标
（1） 资源利用率：使系统中的处理机和其他所有资源尽可能保持忙碌状态，以提高资源利用率。
（2） 公平性：确保所有进程获得合理的 CPU 时间，避免饥饿现象。公平性是相对的，相同类型进程应获得相同服务，不同类型进程可根据紧急程度或重要性提供不同服务。
（3） 平衡性：调度算法应尽可能保持系统资源使用的平衡性，使 CPU 和各种外部设备经常处于忙碌状态。
（4） 策略强制执行：必须准确执行所制定的策略（包括安全策略），即使某些工作可能因此延迟。

二、 批处理系统的调度目标
（1） 平均周转时间短
（2） 系统吞吐量高
（3） 处理机利用率高

2. 在选择调度方式和调度算法时，应遵循的准则是什么?

答：选择调度方式和算法时应遵循以下准则：
• 资源利用率：使 CPU 和其他资源尽可能保持忙碌。
• 公平性：确保每个进程都能获得合理的 CPU 时间，避免饥饿现象。
• 平衡性：使系统中 CPU 和外部设备的使用保持平衡。
• 策略强制执行：确保安全策略等必须被严格执行。
• 系统类型目标：
  - 批处理系统：平均周转时间短、系统吞吐量高、CPU 利用率高。
  - 分时系统：响应时间快、均衡性（响应时间与任务复杂度匹配）。
  - 实时系统：保证截止时间、可预测性。

3. 在批处理系统、分时系统和实时系统中，各采用哪几种进程（作业）调度算法?

答：
• 批处理系统：先来先服务（FCFS）、短作业优先（SJF）、优先级调度、高响应比优先（HRRN）。
• 分时系统：时间片轮转（RR）、多级反馈队列调度。
• 实时系统：最早截止时间优先（EDF）、最低松弛度优先（LLF）、优先级调度（可抢占）。

4. 何谓静态和动态优先级?确定静态优先级的依据是什么?

答：静态优先级：在进程创建时确定，且在整个运行期间保持不变。动态优先级：在进程运行过程中根据情况动态调整。
确定静态优先级的依据：
- 进程类型（系统进程优先级高于用户进程）。
- 资源需求（对资源需求少的进程优先级高）。
- 用户要求（紧迫程度和付费情况）。

6. 在时间片轮转法中，应如何确定时间片的大小?

答：时间片应略大于一次典型交互所需的时间（如几次键盘输入），以确保大多数交互进程能在一个时间片内完成。时间片太小会导致频繁上下文切换，增加系统开销；太大则退化为 FCFS，影响响应时间。通常时间片取 10~100ms。

7. 按调度方式可将实时调度算法分为哪几种?

答：
• 非抢占式调度：
  - 非抢占式轮转调度
  - 非抢占式优先级调度
• 抢占式调度：
  - 基于时钟中断的抢占式优先级调度
  - 立即抢占的优先级调度

8. 什么是最早截止时间优先调度算法?举例说明之。

答：定义：根据任务的截止时间确定优先级，截止时间越早优先级越高。
例如，假设两个周期任务：
- 任务 A：周期 20ms，执行时间 10ms
- 任务 B：周期 50ms，执行时间 25ms
调度器优先调度截止时间更早的任务（如 A1 在 t=0 时截止时间早于 B1），确保错过截止时间的风险最小。

9. 什么是最低松弛度优先调度算法?举例说明之。

答：定义：根据任务的紧急程度（松弛度 = 截止时间 - 剩余运行时间 - 当前时间）确定优先级，松弛度越小优先级越高。
例子：
- 任务 A：截止时间 20ms，剩余运行时间 10ms，当前时间 t=0，松弛度 = 10ms
- 任务 B：截止时间 50ms，剩余运行时间 25ms，松弛度 = 25ms
优先调度 A（松弛度更小）。若在 t=10ms 时 A2 松弛度变为 20ms，B1 松弛度变为 15ms，则切换调度 B1。

10. 通过一个例子来说明通常的优先级调度算法为什么不能适用于实时系统?

答：举例（优先级倒置）：假设三个进程：P1（优先级高）、P2（中）、P3（低）。P3 占用临界资源，P1 等待该资源。但 P2 抢占 P3 的 CPU，导致 P3 无法及时释放资源，P1 被阻塞且等待时间不可预测。
问题：普通优先级调度无法保证高优先级任务在截止时间内完成，缺乏可预测性，不满足实时系统要求。
解决方案：采用优先级继承（如 P3 继承 P1 的优先级）避免被 P2 抢占。
