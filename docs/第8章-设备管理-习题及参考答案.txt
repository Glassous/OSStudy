1. I/O 设备分类依据及方式。

答：I/O 设备可以从多个维度进行分类：
• 按信息组织/传送单位分类：
  - 块设备（如磁盘，数据以字符块为单位传输，支持随机读写）；
  - 字符设备（如打印机、键盘，数据以字符为单位传输，不支持随机读写）。
• 按使用特性分类：
  - 存储设备（如磁盘、光盘）；
  - 输入输出设备（如键盘、显示器）。
• 按设备共享属性分类：
  - 独占设备（同一时间内只允许一个进程使用）；
  - 共享设备（允许多个进程同时/交替使用）；
  - 虚拟设备（通过 SPOOLing 技术将独占设备虚拟为共享设备）。
• 按数据传输速率分类：
  - 低速设备（如键盘，每秒几个到数百字节）；
  - 中速设备（如打印机，每秒数千到数万字节）；
  - 高速设备（如磁盘，每秒数十 KB 到数 MB 以上）。

2. 什么是独占设备、共享设备和虚拟设备？

答：
• 独占设备：是指在同一时刻只允许一个进程占用的设备（如打印机）。一旦分配给某进程，在释放前其他进程无法使用它。
• 共享设备：是指允许多个进程同时或交替访问的设备（如磁盘）。它具有较高的资源利用率。
• 虚拟设备：是指通过 SPOOLing 等通道共享技术，将一台物理上的独占设备改造为若干台逻辑上的共享设备，供多个进程同时“使用”。

3. 设备与设备控制器之间有哪些信号线？并简要说明各自的作用。

答：三类信号线：
• 数据信号线：用于在 I/O 设备和设备控制器之间传送实际的数据信号。
• 控制信号线：用于传送由设备控制器向外设发送的各种控制命令（如启动、读写等控制信号）。
• 状态信号线：用于由外设向控制器传送反映设备当前运行状态的反馈信号（如忙、就绪、故障等状态）。

4. 设备控制器的基本组成及其具备的功能。

答：
• 基本组成：
  - 控制寄存器：接收并保存来自 CPU 的 I/O 命令。
  - 数据寄存器：暂存 CPU 与外设之间要交换的数据。
  - 状态寄存器：保存外设当前的运行状态（如忙、错误），供 CPU 查询。
  - 地址译码器：识别并响应系统分配给该控制器的专属设备地址。
• 主要功能：接收和识别命令；数据交换；设备状态记录与报告；设备地址识别；差错检测与控制等。

5. 什么是 I/O 通道？设置它的作用是什么？

答：
• 定义：I/O 通道是一种特殊的、具有单一指令类型的通道处理器，能够脱离 CPU 独立执行专门的通道指令程序来控制外设操作。
• 作用：为了使 CPU 从繁重的 I/O 控制和日常事务中解放出来。设置通道后，CPU 只需向通道发送一条启动指令，通道就会自动装载并执行内存中的通道程序完成整个 I/O 操作，实现 CPU 与通道、以及多外设之间的全面并行，显著提高系统吞吐量。

6. I/O 通道与一般处理器的比较。

答：
• 相同点：均有自己的指令系统，能独立执行程序并控制数据传输。
• 不同点：
  - 指令单一性：I/O 通道的指令集极其单一且专门化，主要局限于数据传输与 I/O 控制指令，硬件结构远比一般 CPU 简单。
  - 内存共享性：通道没有自己独立的物理内存，它执行的通道程序必须存放在主机的物理内存中，与 CPU 共享系统内存。

7. I/O 控制方式有哪些？并依据 CPU 介入程度对它们进行比较。

答：四种控制方式（按 CPU 介入程度从高到低排列）：
（1） 程序直接控制方式：CPU 必须通过循环轮询（忙等待）的方式不断测试设备控制器状态，直到设备准备就绪才进行一个字（节）的数据传输。CPU 介入度极高，资源浪费严重。
（2） 中断驱动方式：CPU 发出 I/O 命令后可以切换执行其他任务，当设备控制器完成一个字（节）的传输后，向 CPU 发出中断请求，CPU 再介入处理下一个数据。CPU 介入度有所降低，实现字级并行。
（3） 直接存储器存取（DMA）方式：数据直接在内存和设备控制器之间成块传送，不需要 CPU 干预。只有在整块数据传输完成后，控制器才向 CPU 发出一次中断请求。CPU 介入度大幅降低，实现块级并行。
（4） 通道控制方式：进一步减少 CPU 干预。CPU 只需发出一条通道命令启动通道，通道便通过执行通道程序自主管理和控制多个数据块（乃至多个设备）的读写。仅在整个通道程序执行完毕后才发出一次中断。CPU 介入度极低。

8. 试述程序直接控制 I/O 方式的处理流程。

答：以 CPU 从输入设备读取数据为例：
（1） CPU 向输入设备控制器发出读指令，同时将状态寄存器中的 busy 标志置为 1，启动输入设备。
（2） CPU 进入循环轮询状态，不断读取并测试状态寄存器中的 busy 标志。
（3） 当 busy=1 时，表示数据尚未输入完毕，CPU 继续循环等待；当 busy=0 时，表明输入数据已送入控制器的数据寄存器。
（4） CPU 将数据寄存器中的数据取出，写入系统内存的指定单元。
（5） 完成一个字符的 I/O 后，若还需继续读取，CPU 再次启动设备并将 busy 重新置为 1，重复上述步骤。

9. 试述中断 I/O 控制方式的处理流程。

答：流程如下：
（1） CPU 发出启动外设的 I/O 指令，使外设处于准备工作状态。随后，CPU 挂起当前等待 I/O 的进程并进行调度，转去执行其他并发进程。
（2） 设备控制器控制外设进行读写操作，外设与 CPU 保持并行。
（3） 当设备准备就绪（例如一个字节的数据已存入数据寄存器）时，设备控制器向 CPU 发出中断信号。
（4） CPU 接收到中断后暂停当前工作，保护现场，转去执行对应的 I/O 中断服务子程序。
（5） 中断服务子程序将控制器中的数据读入内存，或将内存数据写入控制器。处理完成后，恢复现场，返回被打断的进程继续执行。

10. 说明中断 I/O 控制方式与 DMA 方式有何不同。

答：主要区别如下：
（1） 传输单位：中断 I/O 方式以“字或字节”为单位进行数据传输和中断交互；DMA 方式以“数据块”为单位进行数据传输和中断交互。
（2） CPU 干预度：中断 I/O 方式下，每完成一个字节的传输就需要 CPU 中断干预一次；DMA 方式下，CPU 仅在整个数据块传输开始前（设置传送参数）和传送结束后（响应块完成中断）才介入，数据传送过程完全由 DMA 硬件控制，不占用 CPU 周期。
（3） 数据通路：中断 I/O 方式的数据必须经过 CPU 寄存器中转；DMA 方式下，数据直接在设备和内存之间传输，不需要经过 CPU 寄存器。

11. 试述引入缓冲的目的并比较几种缓冲技术。

答：
• 引入缓冲的目的：缓和 CPU 与 I/O 设备之间速度不匹配的矛盾；减少 CPU 的中断频率，放宽对中断响应时间的限制；提高 CPU 和 I/O 设备之间的并行性。
• 缓冲技术比较：
  （1） 单缓冲：系统在内存中分配一个缓冲区。当用户进程发出 I/O 请求时，外设先将数据填满该缓冲区，进程才能从中读取。由于读写共用一个缓冲，读写操作不能同时进行。
  （2） 双缓冲：系统分配两个缓冲区。在一个缓冲区被输入填满的同时，另一个缓冲区可以被用户进程读取，实现输入和读取的并行。
  （3） 循环缓冲：将多个缓冲区组织成环形队列，包含空缓冲区、已装满缓冲区以及正在使用的缓冲区，适合输入与输出速率大致均衡且高速并发的场景。
  （4） 缓冲池：属于系统级共享缓冲。池中包含多个空缓冲队列、输入队列和输出队列，通过统一的按需分配机制服务于系统中的所有进程，资源利用率最高。

12. 磁盘 I/O 速度与哪些因素有关？如何提高磁盘 I/O 速度？

答：
• 相关因素：
  - 寻道时间（磁头移动到指定磁道的时间，占比最大）；
  - 旋转延迟（磁盘旋转使目标扇区到达磁头下方的时间）；
  - 数据传输时间（磁头读写数据的时间）。
• 提高速度的方法：
  - 采用磁盘调度算法（如 SSTF、SCAN）优化磁头移动轨迹，减少平均寻道时间；
  - 采用多磁盘镜像、RAID（独立磁盘冗余阵列）技术提高并行读写能力；
  - 设置磁盘高速缓存，减少物理磁盘 I/O 的启动频率。

13. 试分别 FCFS、SSFS、SCAN 几种磁盘调度算法的优缺点。

答：
（1） FCFS（先来先服务）：
• 优点：简单、公平，不会发生进程饥饿。
• 缺点：完全未对寻道做优化，磁头来回移动剧烈，平均寻道时间极长。
（2） SSTF（最短寻道时间优先）：
• 优点：每次选择距离当前磁头最近的请求，平均寻道时间大为缩短，吞吐量高。
• 缺点：边缘磁道的请求容易发生“饥饿”现象。
（3） SCAN（电梯调度）：
• 优点：磁头单向移动，依次服务沿途请求，直到边缘才反向。有效克服了 SSTF 的“饥饿”问题，性能也较好。
• 缺点：对刚越过磁头位置的请求响应不及时。

14. 试比较 N-step-SCAN 和 FSCAN 两种调度算法的异同。

答：
• 相同点：两算法均是对 SCAN 和 C-SCAN 的改进，旨在解决“磁臂黏着”现象（即当有源源不断的高频局部磁道请求到达时，磁头停留在某磁道不移动，导致其他远端请求被无限期推迟的现象）。
• 不同点：
  - N-step-SCAN：将磁盘请求队列分成若干个长度为 N 的子队列，按照 FCFS 依次处理每个子队列，而在处理某个队列时，新出现的请求只能放到后续其他队列中，保证了当前扫描的确定性。
  - FSCAN：是 N-step-SCAN 的简化版，只使用两个子队列。当前扫描时，所有请求被放入一个队列中按 SCAN 处理；而在扫描处理期间新出现的全部请求都被推迟，直接放入另一个等待队列，直至下一次扫描时才进行处理。
