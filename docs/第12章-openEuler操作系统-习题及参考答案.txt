1. openEuler 支持哪些指令系统？

答：openEuler 是一款基于 Linux 内核的开源操作系统，支持 CPU（包括 ARM、x86、RISC-V 等多种指令集）、GPU、NPU 等多样性算力，并支持服务器、云计算、边缘计算、嵌入式等应用场景。

2. openEuler 有哪些版本？

答：openEuler 通常有两种版本，一种是创新版本，支撑 Linux 爱好者技术创新，内容较新，如 openEuler 20.09，通常半年发布一个新的版本。另一种是 LTS 版本，是 openEuler 操作系统发行版的稳定版本，如 openEuler LTS 20.03，通常两年发布一个新的版本。

3. openEuler 的体系结构是什么样的？

答：openEuler 操作系统采用五层架构设计，从底层到上层依次为：
• 内核层 作为系统核心，负责硬件资源管理，支持 x86、ARM、RISC-V 等主流处理器架构。
• 系统服务层 提供基础功能模块，如网络、存储、安全等，确保系统稳定运行。
• 平台中间件层 包含容器化系统 KubeOS、分布式文件系统等关键技术，支持云原生业务调度算法，实现多核性能优化。
• 应用开发与运维工具层 提供开发工具链和运维管理功能，简化应用部署与维护流程。
• 用户接口与安全控制层 包含图形界面和安全机制，保障系统安全性和易用性。

4. openEuler 如何定义用户和组？

答：openEuler 是一个多用户的操作系统，所有要使用系统资源的用户需要先向系统管理员申请一个账号，之后用此账号进入系统；可以在系统上建立多个用户，而多个用户可以在同一时间内登录至同一系统执行不同的任务，并不会相互影响。
用户组是具有相同特性用户的逻辑集合。通过组的形式使得具有相同特性的多个用户能够拥有相同的权限，便于管理；每一个用户都拥有自己的私有组；同一组内的所有用户可以共享该组下的文件；每一个用户组都会被分配一个特有的 id 号。
将用户分组是 openEuler 系统中对用户进行管理及控制访问权限的一种手段，通过定义用户组，在很大程度上简化了管理工作。
用户和用户组管理是系统安全管理的重要组成部分，通过操作命令行能够对用户组文件进行创建、修改、删除以及关联用户等操作。
相关命令包括创建用户 useradd 命令、修改用户的密码 passwd 命令、修改用户 usermod 命令、创建组 groupadd 命令、删除组 groupdel 命令等。

5. shutdown 和 halt 命令有什么区别？

答：
• shutdown 命令的作用是关闭计算机，使用权限是超级用户。
• halt 命令的作用是关闭系统，使用权限是超级用户。
• 关机流程差异 shutdown：通过发送系统信号给 init 进程，逐步关闭系统。它会通知所有登录用户、保存数据、同步文件系统，最后关闭电源。halt：直接调用 shutdown -H 停止所有进程和服务，但不会切断电源。
• 系统状态保留 shutdown：默认保留系统状态（如运行级别），可重启或进入单用户模式.halt：停止所有进程但不关闭电源，适用于维护或快速停止系统。
• 命令参数区别 shutdown：支持-r 重启、-h 关机、-n 不使用 init 等选项。halt：通常不单独使用，实际调用的是 shutdown -H。
• 应用场景 shutdown：适用于计划性关机或重启，需保留系统状态。halt：用于快速停止系统（如维护），但需手动断电。

6. 简述 openEuler 的文件目录结构。

答：在 openEuler 中，一切都是文件，其文件目录结构是树形，“/”称为根目录。
“一切皆文件”，是指对所有文件（目录、字符设备、块设备、套接字、打印机、进程、线程、管道等）操作，读写都可用 fopen()/fclose()/fwrite()/fread() 等函数进行处理。
登录系统后，在当前命令窗口下输入命令：ls /
主要目录结构如下：
• /bin：bin 是 Binary 的缩写，这个目录存放着最经常使用的命令。
• /boot：这里存放的是启动 openEuler 时使用的一些核心文件，包括一些连接文件以及镜像文件。
• /dev：dev 是 Device(设备)的缩写，该目录下存放的是 openEuler 的外部设备，在 openEuler 中访问设备的方式和访问文件的方式是相同的。
• /etc：这个目录用来存放所有的系统管理所需要的配置文件和子目录。
• /lib：这个目录里存放着系统最基本的动态连接共享库，其作用类似于 Windows 里的 DLL 文件。几乎所有的应用程序都需要用到这些共享库。

7. 为保证系统安全，openEuler 定义了哪些文件权限？

答：权限是操作系统用来限制对资源访问的一种机制，权限一般分为读（r）、写（w）、执行（x）。在 openEuler 系统中，不同的用户所处的地位也不尽相同，不同地位的用户拥有不同的权限等级，为了保证系统的安全性，openEuler 系统针对不同用户的权限制定了不同的规则。

8. openEuler 软件包工具 DNF 有哪些功能？

答：openEuler 的软件包的工具 DNF，是一款 Linux 软件包管理工具，用于管理 RPM 软件包。DNF 可以查询软件包信息，从指定软件库获取软件包，自动处理依赖关系以安装或卸载软件包，以及更新系统到最新可用版本。
DNF 与 YUM 完全兼容，提供了 YUM 兼容的命令行以及为扩展 and 插件提供的 API，所以在 openEuler 使用命令行的工具中，可以使用 DNF 也可以使用 YUM，根据自己的喜欢来用。在使用 YUM 的时候需要管理员权限，所有命令需要在管理员权限下执行。

9. 安装 Hadoop 为什么要配置免密钥登录？

答：
• 自动化运维需求 Hadoop 集群管理涉及频繁远程操作，例如启动/停止守护进程、数据迁移等。若每次操作均需手动输入密码，将极大降低运维效率。
• 安全性保障 SSH 协议采用加密传输，可有效防止数据泄露、中间人攻击等安全威胁。通过公钥认证机制，即使密钥文件被窃取，攻击者也无法直接获取私钥。
• 分布式系统特性 Hadoop 集群由多台机器组成，免密钥登录支持跨节点无密码验证。
• 简化集群部署 在完全分布式模式下，免密钥登录可避免因网络延迟或手动输入错误导致的部署失败。尤其在虚拟机或容器化环境中，自动化脚本依赖此功能实现快速部署。

10. 如何测试 Hadoop 是否成功启动？

答：通过 18088 端口可以查看集群相关信息，在浏览器中输入网址 http://master:18088，可以看到集群状态、节点是否启动等。
验证安装可以运行 hadoop 自带的实例。如执行实例程序计算 PI，在目录 /opt/hadoop-3.1.3/share/hadoop/mapreduce，执行结果可以看出集群工作是否正常。
